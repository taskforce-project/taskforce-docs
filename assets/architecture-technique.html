🔄 Cycle d'une requête

  1. Login → Keycloak émet le JWT.
  2. Front appelle /api/** (Bearer).
  3. Backend valide le JWT (JWKS), vérifie l'accès (workspace/projet).
  4. Service → repository → PostgreSQL.
  5. Réponse ApiResponse<T> → data.data.

⚡ Temps réel

  • STOMP via RabbitMQ (/topic/channel.{id}).
  • Repli SimpleBroker en mémoire si RabbitMQ absent.
  • Auth STOMP au CONNECT (interceptor).

🧭 Dev vs Prod

  • dev compose complet + pgAdmin, ports exposés, Keycloak buildé.
  • prod Nginx+TLS, Keycloak 23, secrets via env, ports internes.

🔐 Sécurité

  • JWT RS256 · CORS · CSRF off (stateless).
  • Chaînes publiques (/api/auth,/stripe,/files) vs JWT.
  • ⚠️ pas de context-path → préfixe /api obligatoire.
