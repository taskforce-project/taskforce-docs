## 1. Section principale

<p class="lead">
Paragraphe d'introduction qui présente le contexte de manière narrative et engageante. Ce paragraphe doit capter l'attention du lecteur et lui donner envie de poursuivre sa lecture. Il est rédigé de manière fluide, sans liste à puces.
</p>

</div>

### 1.1 Sous-section

Le contenu de cette sous-section est rédigé sous forme de paragraphes cohérents. Plutôt que d'énumérer des points de manière sèche, on développe les idées en les reliant entre elles. Par exemple, lorsqu'on aborde plusieurs aspects d'un même sujet, on les présente dans un flux narratif naturel, en utilisant des transitions comme "par ailleurs", "de plus", "en outre", ou "parallèlement à cela".

<blockquote class="important">
<strong>Point clé :</strong> Les informations critiques peuvent être mises en évidence dans des encadrés comme celui-ci, pour attirer l'attention du lecteur sans rompre le flux de lecture.
</blockquote>

Lorsqu'il est nécessaire de présenter plusieurs éléments connexes, on peut les intégrer directement dans le texte en les séparant par des virgules ou des points-virgules. Par exemple, le projet mobilise plusieurs compétences techniques : le développement front-end avec React et TypeScript, le développement back-end avec Spring Boot et Java, la gestion d'infrastructure avec Docker, ainsi que la mise en place de pipelines CI/CD avec GitHub Actions.

<div class="highlight-box">

**Cas particulier :** Lorsque les informations sont trop nombreuses ou complexes pour être présentées en paragraphe, un tableau reste approprié :

| Critère | Description   | Exemple |
| ------- | ------------- | ------- |
| A       | Description A | Ex A    |
| B       | Description B | Ex B    |

</div>

### 1.2 Utilisation du HTML

Le HTML permet d'enrichir la présentation sans compromettre la lisibilité :

<div style="background-color: #f8f9fa; padding: 1rem; border-left: 4px solid #007bff; margin: 1rem 0;">
<strong>Exemple d'encadré stylisé</strong><br>
Ce type d'encadré peut être utilisé pour mettre en évidence des informations importantes, des exemples concrets, ou des recommandations spécifiques.
</div>

Les listes ne sont utilisées que lorsqu'elles sont véritablement nécessaires, par exemple pour des énumérations courtes et factuelles où un paragraphe nuirait à la clarté. Dans ce cas, on privilégie des listes courtes avec des items concis.

## 2. Autre section

<p class="lead">
Cette section démontre comment alterner entre différents styles de présentation selon le contenu. L'important est de privilégier la narration tout en maintenant une clarté d'expression et une structure logique.
</p>

Le texte se poursuit de manière fluide, en développant les arguments progressivement. Chaque paragraphe introduit une idée, la développe, puis établit une transition vers le paragraphe suivant. Cette approche rend la lecture plus agréable et facilite la compréhension globale du document.
